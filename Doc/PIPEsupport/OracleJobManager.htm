OracleJobManager Record

The OracleJobManager record in the PanCIM database is responsible for activating the interface program to read the Oracle job packet table and make any necessary updates to either database. The frequency of the activations can be adjusted by changing the reschedule interval field value. The specific organization ID and department codes read by and processed by the PIPE program are determined by entries in this record.

Field Name

Description

LOGS_IN_MEMORY

Number of logged status occurrences

#_DEPARTMENTS

Number of departments to include in Oracle job record retrieval

DB_NAME

Oracle database instance

DB_USER

Oracle database user name

DB_PWD

Oracle database password

OrganizationCode

Oracle code for division (see Division Codes)

OrganizationID

Oracle ID for division (see Division Codes)

Scheduled_Time

Indicates next record activation time

Reschedule_Interval

Interval between record activations

Current_Status

TSK_PIPE will update this field with last status (see Job Manager Status codes)

Activation_Type

Not currently used

Records_Added

Number of records added to PanCIM database during last activation

Records_Modified

Number of records modified during last activation

Records_Removed

Number of records removed from the PanCIM database during last activation

Log_Time

Time of log entry

Added_Count

Logged value for number of records added

Modified_Count

Logged value for number of records modified

Removed_Count

Logged value for number of records removed

Status

Logged status code

Department_ID

Oracle department ID to include in the record retrieval

 

IMPORTANT: TSK_PIPE only reads information from this record when initially starting. If changes to this record are made while TSK_PIPE is running, it needs to be stopped and restarted in order to refresh it's data.
