Trouble Shooting

Listed below are a few tips that may help to determine the cause of a suspected problem with the Production Information PanCIM Exporter program.

 

The first action for any suspected problem with this program should be to (1) make sure it's actually listed as running in the InfoPlus.21 Manager and (2) if it is, find out if it's responding to events.

If it's not running, start it. The first place to check if the program is responding is to look at the OracleJobManager record in the InfoPlus.21 database.

The CURRENT_STATUS field should contain OK and the LAST_LOG_TIME should have a timestamp no less than the current time minus the RESCHEDULE_INTERVAL value. Unless there is a large amount of Oracle job records to process, you should never see the text Processing in the CURRENT_STATUS field. If you do, the program is "hung" and will need to be shut down through the Microsoft Task Manager and restarted through the InfoPlus.21 Manager.

The program also writes the count of added, modified and removed records in the appropriate fields after every activation. The LOGS_IN_MEMORY repeat area should have two entries for each activation interval with a recent LOG_TIME timestamp. Each activation time should have a status entry of Processing and then OK.

 

Another source for troubleshooting other suspected problem is the PIPE.LOG file.

Several events are recorded in this file. Most of the entries are simply informational but all warnings and/or errors detected by the program are prefixed with *** in the left margin. In the sample above there is a warning that job 11343825 was completed by Oracle but is still listed as running in PanCIM. This can occur when someone transacts a production quantity in Oracle that satifies the planned quantity and does not stop the order in PanCIM.
