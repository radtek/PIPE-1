This condition may have several causes.

  • The Organization ID or code in the OracleJobManager record is incorrect for the division. Enter the correct information and re-start TSK_PIPE.

TSK_PIPE is running and OracleJobManager logging is updating but new workorder or job records do not appear in InfoPlus.21 database after printing in Oracle.

The best first resource for troubleshooting any suspected problems with this application is the PIPE.LOG file.

Many events are recorded in this file. Most of the entries are simply informational but all warnings and errors detected by the program are prefixed with *** in the left margin.

In the sample above there is a warning that job 11343825 was completed by Oracle but is still listed as running in PanCIM. This can occur when someone transacts a production quantity in Oracle that satisfies the planned quantity but does not stop the order in PanCIM. The InfoPlus.21 workorder or job record will not be updated with the Oracle quantity until the job is STOPPED in PanCIM. Since the Oracle flags are not reset in this condition, TSK_PIPE will attempt to perform the necessary updates on the next activation and continue until successful.

This log file will also contain any MSSQL and/or Oracle ODBC driver errors if the occur.
