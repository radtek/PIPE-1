PanCIM Message Record

Several PanCIM applications use the Production Information PanCIM Exporter program to update various relational databases and tables through records created under the PanCIMMessageDef definition record. The PIPE program processes these messages based on the code in the TransCode field. After the data has been successfully transferred, the message record is deleted.

There are two special case records defined by PanCIMMessageDef that should always be present. When these records are activated, TSK_PIPE performs various maintenance functions. (see Scheduled Activations)

Value

PanCIMMsgCheck

ReActivate

17

Re-activates any records defined by PanCIMMessageDef for processing in the event they did not initially due to some type of problem with the Oracle database connection. The activation schedule is contained in the ActPanCIMMsg record. (see Scheduled Activations)

ActWOCleanup

WO Cleanup

18

When activated, TSK_PIPE will search for and delete any record defined by WorkorderDef that meets the current rules. The current criteria for deletion is that the WOStatus field must contain the REMOVE code and the DateDone field timestamp must be greater than 60 days from the current date. The activation schedule is contained in the ActWoCleanup record. (see Scheduled Activations)

 

The values for the TransCodes are contained in the PanCIMMsgTypes record.
