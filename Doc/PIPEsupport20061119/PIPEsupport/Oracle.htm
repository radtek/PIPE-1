Oracle Tables

The Production Information PanCIM Exporter program reads data from several fields in the APPS.PANDUIT_WIP_PANCIM_JOB_V table in the Oracle database. This table contains the job information packet for all Panduit divisions but only records that satisfy the information contained in the OracleJogManager record are retrieved on each activation.

The PIPE program processes the Oracle job data based on the contents of the six flag fields and/or the INTERFACE_STATUS code found in each record retrieved. When new jobs are created, Oracle creates a new record in this table with the job data, sets the INTERFACE_STATUS field to "NEW" and sets all six flag fields to "Y". After TSK_PIPE successfully processes the record, it returns "OK" to the INTERFACE_STATUS field and sets all six flags to "N".

When data in any of the six flagged fields is changed, Oracle sets the associated flag field "Y". TSK_PIPE reads and processes any record in this table that meets the filtering established in the OracleJobManager record and has at least one of the flag fields in this table set to "Y". After successfully updating the necessary PanCIM and/or MSSQL tables, it resets the associated Oracle flag field back to "N".

Note: Since the APPS.PANDUIT_WIP_PANCIM_JOB_V table is actually a view, TSK_PIPE does not write to this table directly. It uses the WIP_ENTITY_ID stored in the InfoPlus.21 workorder or job record to locate and update the correct row in the APPS.PANDUIT_WIP_PANCIM_JOB table.
